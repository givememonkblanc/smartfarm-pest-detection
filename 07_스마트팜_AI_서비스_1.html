3 객체 탐지 모델의 기본 이해

병해 탐지의 핵심 기술은 "객체 탐지(Object Detection)"입니다. 이미지 분류와 어떻게 다른지, YOLO가 무엇을 출력하는지 이해합니다.

3.1 객체 탐지란?

🏷️ 이미지 분류 (Classification)

"이 사진은 병든 잎이다" — 사진 전체에 라벨 하나. 위치는 모릅니다.

📦 객체 탐지 (Detection)

"여기, 여기, 여기에 병해 부위가 있다" — 위치(박스) + 클래스 + 신뢰도를 함께 출력.

객체 탐지는 이미지 안에서 다음 3가지를 동시에 알려줍니다.

  • 객체 위치 탐지 — 바운딩박스 좌표 (x1, y1, x2, y2)
  • 클래스 예측 — 그 박스가 어떤 병해인지 (예: leaf_blast)
  • 신뢰도 점수(confidence) — 그 예측을 얼마나 확신하는지 (0~1)

3.2 YOLO 기반 탐지 방식

YOLO(You Only Look Once)는 이미지를 한 번만 보고 모든 객체를 동시에 찾아내는 빠른 객체 탐지 모델 계열입니다. 속도가 빨라 스마트팜·로봇·실시간 영상에 널리 쓰입니다.

inference(추론) 결과 구조

출력 항목의미예시
바운딩박스객체를 감싼 사각형 좌표(120, 80, 240, 210)
클래스 이름탐지된 대상의 종류leaf_blast (도열병)
confidence score예측 신뢰도 (0~1)0.87

하나의 이미지에서 YOLO는 위 묶음을 객체 개수만큼 출력합니다. 즉, 잎 한 장에 병징이 3곳이면 3개의 (박스+클래스+신뢰도) 결과가 나옵니다.

3.3 탐지 결과 해석 방법

  • 탐지 클래스 확인 — 어떤 병해로 분류되었는지 본다.
  • 신뢰도 기준 설정 — 예: 0.5 미만은 무시. 기준이 낮으면 오탐↑, 높으면 놓침↑.
  • 중복 탐지 이해 — 같은 부위에 박스가 겹쳐 나올 수 있음(NMS로 정리).
  • 잘못된 탐지(오탐) 판단 — 물방울·그림자를 병으로 인식하는 경우를 사람이 검증.
실무 팁: "신뢰도 임계값(confidence threshold)"은 서비스 성격에 맞춰 정합니다. 병해를 절대 놓치면 안 되면 낮게, 오탐 알림이 부담되면 높게 설정합니다.

from ultralytics import YOLO

# ① 모델 로드 (가중치 파일 불러오기)
model = YOLO("weights/rice_disease.pt")

# ② 이미지 한 장에 대해 병해 부위 탐지 실행
results = model("images/leaf01.jpg", conf=0.5)

# ③ 탐지 결과 요약 출력
for box in results[0].boxes:
    cls_id = int(box.cls[0])
    name   = model.names[cls_id]
    conf   = float(box.conf[0])
    print(f"탐지: {name}  신뢰도: {conf:.2f}")
conf=0.5 는 신뢰도 0.5 미만 탐지를 버린다는 의미입니다. 결과가 너무 적으면 낮추고, 오탐이 많으면 높이세요.

6 탐지 객체 좌표 계산

탐지 박스의 좌표를 이해하고, 로봇 작업에 필요한 "중심좌표"를 계산합니다.

6.1 바운딩박스 좌표 이해

YOLO 박스는 보통 (x1, y1, x2, y2) 형태입니다.

  • (x1, y1) — 박스의 좌상단 모서리
  • (x2, y2) — 박스의 우하단 모서리
이미지 좌표계: 화면 왼쪽 위가 원점(0,0)입니다. x는 오른쪽으로, y는 아래로 증가합니다(수학과 반대!). 이 "좌상단 기준 체계"를 기억하세요.
 (0,0) ───────────► x 증가
   │   ┌───────────┐
   │   │(x1,y1)    │
   │   │   병해     │
   │   │    부위    │
   │   │    (x2,y2)│
   │   └───────────┘
   ▼
 y 증가

6.2 중심좌표 계산

로봇이 "어디로 가야 하는가"는 박스 모서리가 아니라 중심점으로 판단하는 것이 자연스럽습니다.

# 중심좌표 계산 공식
cx = (x1 + x2) / 2      # 중심 x
cy = (y1 + y2) / 2      # 중심 y
center.py 
for box in results[0].boxes:
    x1, y1, x2, y2 = box.xyxy[0].tolist()   # 좌표 4개
    cx = (x1 + x2) / 2
    cy = (y1 + y2) / 2
    print(f"중심점: ({cx:.0f}, {cy:.0f})")

이 중심점을 결과 이미지에 점으로 찍어 시각화하면 로봇이 겨냥할 위치를 직관적으로 보여 줄 수 있습니다.

7.2 JSON 파일 저장

to_json.py 
import json
from datetime import datetime

detections = []
for box in results[0].boxes:
    x1, y1, x2, y2 = box.xyxy[0].tolist()
    cls_id = int(box.cls[0])
    detections.append({
        "class": model.names[cls_id],
        "confidence": round(float(box.conf[0]), 3),
        "bbox": [round(x1), round(y1), round(x2), round(y2)],
        "center": [round((x1+x2)/2), round((y1+y2)/2)],
    })

output = {
    "image": "leaf01.jpg",
    "time": datetime.now().isoformat(timespec="seconds"),
    "detections": detections,   # 여러 객체 결과를 리스트로
}

with open("results/leaf01.json", "w", encoding="utf-8") as f:
    json.dump(output, f, ensure_ascii=False, indent=2)

8 모델 선택 및 가중치 교체

같은 코드로 다른 작물·다른 객체를 탐지하려면 "가중치 파일"만 바꾸면 됩니다. 그 의미를 이해합니다.

8.1 YOLO 가중치 파일의 역할

  • 모델 구조 + 가중치 — .pt 파일에는 신경망 구조와 학습된 숫자(가중치)가 담겨 있습니다.
  • 학습된 클래스 정보 — 어떤 대상을 탐지하도록 학습됐는지가 함께 저장됩니다.
  • 가중치 교체의 의미 — 코드는 그대로 두고 .pt 파일만 바꾸면 탐지 대상이 바뀝니다.

8.2 모델별 클래스 정보 확인

# 어떤 대상을 탐지할 수 있는지 확인
print(model.names)          # 클래스 이름 목록
print(len(model.names))     # 클래스 수
  • 클래스 이름 확인 — 모델이 무엇을 아는지
  • 클래스 수 확인 — 몇 종류를 구분하는지
  • 탐지 가능 대상 확인 — 목록에 없는 대상은 절대 탐지 못함

12 GitHub로 코드 저장소 관리

만든 코드를 안전하게 보관하고, 변경 이력을 남기며, 다른 곳에서도 받아 쓸 수 있게 GitHub 저장소(repository)로 관리합니다.

12.1 Git·GitHub 기초

용어의미
Git코드의 변경 이력을 기록·관리하는 버전 관리 도구
GitHubGit 저장소를 인터넷에 올려두는 온라인 보관·공유 공간
repository(repo)한 프로젝트의 코드와 이력이 담긴 저장소

지금까지 배운 것을 모두 연결합니다. OpenCode로 웹앱을 만들고 → GitHub 저장소로 관리하여, 7회차 벼 병해 탐지 기능을 하나의 웹 서비스로 완성합니다.

13.1 프로젝트 구성

project structure 
rice-disease-web/
 ├─ app.py            # 웹앱 메인 (Streamlit)
 ├─ detect.py         # 탐지·좌표·JSON 로직 (5~7장)
 ├─ weights/          # 가중치 (.gitignore 처리)
 ├─ images/           # 테스트 이미지
 ├─ requirements.txt  # 필요한 라이브러리 목록
 └─ README.md         # 서비스 설명

13.2 OpenCode로 웹앱 생성

프로젝트 폴더에서 opencode를 실행하고, 아래처럼 단계별로 요청합니다.

① 골격 만들기

"detect.py 의 탐지·JSON 함수를 그대로 쓰는 Streamlit 앱 app.py 를 만들어줘.
 업로드 → 탐지 실행 버튼 → 결과 이미지 + JSON 표시 흐름으로."

② 모델 선택 기능 추가

"상단에 모델 선택 드롭다운을 추가하고, 8.3장의 MODELS 딕셔너리와 연결해줘."

③ 의존성 정리

"requirements.txt 에 ultralytics, streamlit 을 추가하고 README 도 작성해줘."

각 단계마다 OpenCode가 만든 코드를 직접 실행해 확인합니다.

terminal 
pip install -r requirements.txt
streamlit run app.py        # 브라우저에서 웹앱 확인
